# Modeling

This section of the documentation is designed for business analysts using A12 but is also valuable for developers looking to understand how models are created. If you’re new to A12, we recommend starting with the [Video introduction to Modeling in A12](#_video_introduction_to_modeling_in_a12).

Here is a brief explanation of what you can find in this section of the documentation.

**Accessing A12**

This section includes two concise guides: the [Quick Start Guide](https://geta12.com/docs/overall/quick_start_guide/index.html), which provides an overview for getting started with the A12 Modeling Environment, and [Installing A12](https://geta12.com/docs/overall/installing_a12/index.html), which offers step-by-step instructions for installation.

**Modeling Environment**

This section provides a comprehensive guide to the core tools of the A12 Modeling Environment: the [Model Editor (SME)](https://geta12.com/docs/sme/sme-ba-docs/index.html), for creating and editing models, and the [Preview App](https://geta12.com/docs/overall/relationships_for_bas/index.html), for testing and refining your applications in a local environment. Additionally, the manual for the [Cloud Modeling Environment](https://geta12.com/docs/overall/cloudmodelingenv/index.html) can be found here.

**Modeling**

This section begins with a [Model Overview](https://geta12.com/docs/overall/types_of_models/index.html), which introduces the various model types created and edited in SME, followed by a guideline for modelers on [Model Naming Conventions](https://geta12.com/docs/overall/model_naming_conventions/index.html) to help ensure consistency and clarity.

The ***Data Modeling*** subsection provides detailed documentation for model types and tools, including:

* [Document Modeling](https://geta12.com/docs/sme/sme-dm-ba-docs/index.html) and [Relationship Modeling](https://geta12.com/docs/overall/relationships_for_bas/index.html).
* Guides for [Kernel Language](https://geta12.com/docs/kernel/kernel-documentation-ba-en/index.html), [Modeling Heterogeneous Data](https://geta12.com/docs/overall/heterogeneity/index.html) and using the [Model Graph Diagram](https://geta12.com/docs/sme/sme-diagram-ba-docs/index.html).
* [Combination Modeling](https://geta12.com/docs/sme/sme-cmm-ba-docs/index.html) and [Mapping Modeling](https://geta12.com/docs/sme/sme-mm-ba-docs/index.html)
* [Query Modeling](https://geta12.com/docs/sme/sme-qm-ba-docs/index.html) and [Workspace Data](https://geta12.com/docs/sme/sme-data-ba-docs/index.html).

The ***UI Modeling*** subsection provides documentation about our UI model types and their associated tooling, including:

* [Form Modeling](https://geta12.com/docs/sme/sme-fm-ba-docs/index.html), [Overview Modeling](https://geta12.com/docs/sme/sme-om-ba-docs/index.html) and [Tree Modeling](https://geta12.com/docs/sme/sme-tm-ba-docs/index.html).
* A dedicated guide for the modeling of relationship UI components, described in [Form Modeling - Binding](https://geta12.com/docs/sme/sme-binding-ba-docs/index.html).
* Documentation on [Content Modeling](https://geta12.com/docs/content_engine/contentengine-user-docs/index.html).

This section also includes guides for advanced topics, such as [Workflow Modeling](https://geta12.com/docs/workflows/ba-docs/index.html), [Application Modeling](https://geta12.com/docs/sme/sme-am-ba-docs/index.html) and using the [Print Model Editor](https://geta12.com/docs/print_engine/print-modeling-documentation/index.html) to design and manage formatted document outputs.

The Preview App, as part of the Modeling Environment, offers comprehensive features such as [Modeling Support for Role Based Access Control](https://geta12.com/docs/overall/comprehensive_features_rbac/index.html) for managing users and roles of an application. Additionally, modelers can leverage the specialized model type described in [Master Detail Modeling](https://geta12.com/docs/sme/sme-mdmm-ba-docs/index.html).

**Tutorials**

Last but not least, step-by-step tutorials are available to help you learn how to model some of the more complex features of A12.

## Video introduction to Modeling in A12

This video is a short introduction to the A12 platform and outlines the benefits of modeling in general. Our aim is to give a brief outline of some of the features that are available in A12 to modelers and business analysts and to show you what you can achieve through modeling alone with A12. We show these features with the help of one of the sample workspaces that come with the A12 installer. This application is fully modeled without any additional development.

This introduction targets everyone who is curious about A12 and would like to get some insight into the possibilities that it offers.

If you are looking for more videos like that, we recommend checking the [Trainings Overview](https://geta12.com/#/trainings/trainings-overview) on getA12.com where you can find a collection of [A12 Introduction Videos](https://geta12.com/#/trainings/trainings-overview).
