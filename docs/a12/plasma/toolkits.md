# Toolkit

## Toolkit for the Visualization of A12 Applications

**For:** Business analysts, project leads, technical project leads, UI/UX designer

**About:** Visualizing use cases, navigation structures and the look and feel of A12 applications as a solid basis for communication with customers and developers.

**Duration:** self-paced

![Presenting visuals of an A12 application](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/hero_image.png)

In the planning phase of a project, it is often difficult to clarify various aspects of an application by expressing them in words only.
With this tutorial and the included PowerPoint libraries, you will be able to create helpful visuals of features, screens and interaction behaviour. These will allow you to have more efficient discussions with customers and/or developers about technical feasibility and implementation.

**How:**

|  |  |  |  |
| --- | --- | --- | --- |
| User flows | Application frame | Content | Theming |
| Show the **user flows** through the application | Mock the **application frame** and **navigation** of your application | Use **mockups** and **example screens** for content discussions | Talk about **A12 themes** with the help of example screens |

## Translating Use Cases into User Flows

This chapter shows how application use cases can easily be visualized with our User Flow library.
The resulting user flows help to quickly visualize the sequences of pages and enable a better overview of the general structure of a website.

![User Flows Library](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/user_flow_library.png)
[Download User Flow Library](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/UserFlowLibrary.pptx)

### Creating User Flows

Once you have derived your use cases from the set of application requirements, you can start to create user flows with the help of the User Flow Library.

Copy the whole slide or single elements from the library and paste them into your target presentation. Now you can adapt them to your user flow step by step (e.g. from Login to Form).

*Example:*

![User Flows](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/user_flows1.png)

If you want to visualize more complex processes or whole sitemaps, you can cluster pages and/or use different colors for the arrows:

![Clustered User Flows](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/user_flows2.png)

You can also use page references, meaning that you add a link to a page cluster which leads to another slide which shows the pages of the cluster in more detail:

![User Flows with page references](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/user_flows3.png)

### Assigning Thumbnails

As soon as you know what kind of pages the individual steps of your user flow will be, you can copy the necessary thumbnails from the user flow library and assign them to the individual steps.
These thumbnails are simplified depictions of our typical A12 views which help to imagine the look and feel of the application without already committing to special details.

*Example:*

![User Flows with Thumbnails](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/user_flows_thumbnails.png)

### Setting up the application frame

|  |  |
| --- | --- |
|  | The structure of the application frame is set in the A12 client and cannot be modified. |

With the application requirements and the user flows, you can now start to set up the application frame, which consists of the application header and optionally the footer:

![Application Frame](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/application_frame1.png)

#### Application Header

The application header is used for global application-wide functionalities and typically consists of the following features:

* Logo
* Application Name
* Global Search
* User Account features
* Login / Logout

![Application Header](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/application_frame2.png)

The header is not restricted to the shown features but in fact provides four slots which can be used for placing information or functionalities.
In case you want to place more then four features we recommend grouping these functionalities to keep the number of options at four.

#### Application Footer

The application footer is not needed for every application. It usually contains information such as imprint, copyrights etc. and/or links to static pages such as "Contact", "Terms of Use", "Accessibility".

### Defining the Navigation Structure

![Navigation structure](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/navigation_structure.png)

#### Main Navigation

If you want to define the navigation structure of an application, the first step is to single out the major modules of your application.
Usually these are only a few, as the complexity lies within the subsections.
These major modules form the main navigation, which takes up the horizontal space at the top of the page:

![Main navigation](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/main_navigation.png)

#### Secondary Navigation

For the navigation points within a major module, it is best to use the vertical secondary navigation on the left side of the page.

For the secondary navigation, there are the following options:

**Flyout Menu**

The flyout menu is like a vertical version of the main navigation menu and consists of a list of navigation links:

![Flyout menu](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/flyout_menu.png)

**Accordion**

Similar to the flyout menu, the accordion is a vertical list of navigation links. The difference is that in the accordion you can collapse or expand the navigation nodes:

![Accordion](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/accordion.png)

**Tab Panel**

The tab panel consists of a list with tabs and a content container.
When a tab is selected, the related content is displayed in the tab panel. This allows quick switching between various related navigation points.

![Tab Panel](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/tab_panel.png)

#### Content Navigation

The content navigation is located in the content area and contains navigation entries which are directly linked to the displayed content.
These navigation entries should be reduced to a minimum and kept simple without a deeper navigation hierarchy.

The content navigation can have a vertical or a horizontal orientation similar to the main and secondary navigation.

*Examples:*

![Content navigation examples](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/content_nav.png)

However, in our Content and Theming library we offer detailed mockups of the different content views with the features that are available.

## Theming

As you might already know, we have four standard design themes: Default, Compact, Flat and Flat Compact.

![Theming overview](https://geta12.com/docs/2025.06/ext5/plasma/toolkits/assets/toolkit/theming.png)

The differences between them is explained in the Content and Theming library and demonstrated on one example view.
We also provide two examples for customized themes in order to show what else is possible apart from the standard.

You can copy these slides also into your presentation to give better examples of how applications can look like with A12.
