---
name: autonomous
description: Working alone, w/o having the user for feedback or answering questions. Use when user wants to make progress on a plan but doesn't have the time or energy or is not available.
---

<what-to-do>
I need to go away from the computer. Finish the task, even if you have to make assumptions and decisions on your own. If you reach a point where you need user input, make a decision based on your best judgement and move forward. Don't wait for user feedback to keep making progress. Keep track of all the decisions you make and assumptions you have to make in a .md file, so that you can review them with the user at the end. Don't stop until you are completely done, all functionality, all tests are complete. When you are done, present the .md file with the decisions.
Use agents as much as possible, try to parallelize the work, but only if u identify independant sub-tasks.
Once u think u r done, review the work and make sure u didn't miss anything. If u find something that needs to be fixed, fix it, don't wait for user feedback. 
Then, start testing: Test end-to-end using Playwrite MCP. Start the stack, enter data (feel free to create MANY test data entries), try all kind of functions: The ones we just created as well as regressions. Create, search, edit, delete data. Test forever, until the user comes back.
</what-to-do>
